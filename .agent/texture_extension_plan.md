# Texture & Normal Map Extension Plan

1. **Primitive material model**: Extend `PrimitiveProperties` (and related data path) to hold per-primitive texture metadata (albedo, normal, enable flags, normal strength). Introduce CPU-side material description that maps to GPU resources per primitive.
2. **Texture resource abstraction**: Implement a `Texture2D` RAII wrapper for sampled images (colour in `VK_FORMAT_R8G8B8A8_SRGB`, normal in `VK_FORMAT_R8G8B8A8_UNORM`) handling image creation, memory allocation, image views, and samplers with consistent filtering/addressing. Provide staged uploads from CPU data.
3. **Descriptor & pipeline wiring**: Update descriptor set layouts/pool so set 0 retains global UBO + shadow map, while set 1 supplies per-primitive textures (albedo + normal). Ensure `GraphicsPipeline` layout creation accepts both set layouts and command recording binds each primitive’s descriptor set prior to draw.
4. **Procedural fallbacks**: Add generators for checkerboard albedo (sRGB) and tangent-space normal noise (unit normals with adjustable seed) used whenever a primitive lacks supplied textures. Cache reusable textures to avoid redundant GPU allocations.
5. **Shader updates**: Modify vertex shader to pass tangents/bitangent axes and fragment shader to sample albedo/normal textures, build TBN matrix, apply normal mapping with strength control, and fall back to vertex normal when textures disabled. Maintain Blinn-Phong workflow and shadow integration.
6. **Runtime integration & UI**: When primitives are (re)initialised, upload textures + descriptor sets; regenerate if properties change. Extend ImGui controls to toggle texture usage, adjust normal strength, and surface texture status. Ensure cascaded shadow previews and toggles remain unaffected.
7. **Validation & docs**: Add unit coverage for procedural texture generation and material state transitions, update integration tests if needed, document new material system and controls in README/docs, and refresh build instructions. Verify build + tests.
